# PyVax Verification Guide

> **Transform Python contracts → Verified Solidity on Snowtrace/Etherscan**

## Overview

PyVax Transformer generates **flattened, single-file Solidity** from your Python contracts — ready for explorer verification.

```
Python Contract → AST Analysis → Flattened Solidity → solc Compile → Deploy → Verify
```

## Quick Start

### 1. Transform Python → Solidity

```bash
# Generate .sol file from Python contract
pyvax transform contracts/Token.py

# Custom output path
pyvax transform contracts/Token.py -o build/Token.sol

# Override contract name
pyvax transform contracts/Token.py --name MyToken
```

### 2. Deploy & Verify

```bash
# Deploy first (standard pyvax flow)
pyvax deploy Token --chain fuji

# Then verify with the deployed address
pyvax transform contracts/Token.py --verify --address 0x1234... --chain fuji
```

### 3. API / Playground

Send a JSON request to the API wrapper:

```json
{
  "command": "transform",
  "source": "from pyvax import Contract...",
  "contract_name": "MyToken"
}
```

Response includes:
- `solidity` — Flattened .sol source
- `abi` — JSON ABI  
- `compiler_version` — solc version used
- `optimization_runs` — Optimizer settings

## What Gets Generated

For a Python contract like:

```python
class Counter(Contract):
    count: int = 0

    @action
    def increment(self):
        self.count = self.count + 1
        self.emit("Incremented", self.msg_sender(), self.count)

    @view_function
    def get_count(self) -> int:
        return self.count
```

The transformer produces:

```solidity
// SPDX-License-Identifier: MIT
// Generated by PyVax Transformer
pragma solidity ^0.8.24;

contract Counter {
    uint256 public count;

    event Incremented(address indexed param0, uint256 param1);

    function increment() external {
        count = count + 1;
        emit Incremented(msg.sender, count);
    }

    function get_count() external view returns (uint256) {
        return count;
    }
}
```

## Type Mapping

| Python | Solidity |
|--------|----------|
| `int` | `uint256` |
| `str` | `address` |
| `bool` | `bool` |
| `dict` | `mapping(address => uint256)` |

## Supported Features

| Feature | Status |
|---------|--------|
| State variables (typed) | ✅ |
| Mappings | ✅ |
| Events with `self.emit()` | ✅ |
| Constructor (`__init__`) | ✅ |
| `@action` functions | ✅ |
| `@view_function` | ✅ |
| `self.require()` | ✅ |
| `self.msg_sender()` / `self.msg_value()` | ✅ |
| `if/else`, `while`, `for range()` | ✅ |
| Comparison / arithmetic operators | ✅ |
| Initial values in declarations | ✅ |
| Snowtrace API verification | ✅ |

## Environment Variables

| Variable | Purpose |
|----------|---------|
| `SNOWTRACE_API_KEY` | Required for `--verify` on Snowtrace |
| `ETHERSCAN_API_KEY` | For Etherscan-compatible chains |

## Architecture

```
avax_cli/
├── transformer.py    # SolidityGenerator + python_to_verified_solidity()
├── snowtrace.py      # SnowtraceVerifier (API client)
├── cli.py            # `pyvax transform` command
└── api_wrapper.py    # `transform` command for playground
```

- **`transformer.py`**: Core engine. Uses `_extract_class_info()` for accurate type/default extraction independent of the analyzer, then `SolidityGenerator` to produce the .sol output.
- **`snowtrace.py`**: HTTP client for Snowtrace/Etherscan verification APIs. Supports Fuji, Mainnet, and custom chains.
